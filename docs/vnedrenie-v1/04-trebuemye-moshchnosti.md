# Требования к вычислительным ресурсам и ПО

Документ для согласования с ИТ-отделом Заказчика. Конкретные цифры зависят от выбранной языковой модели и одновременного числа пользователей; ниже — **ориентиры для версии 1**.

---

## 1. Сервер приложения и база данных

Рекомендуется **одна виртуальная машина** или физический сервер под:


| Ресурс | Минимум (ориентир)                         | Рекомендуется                                                    |
| ------ | ------------------------------------------ | ---------------------------------------------------------------- |
| CPU    | 4 ядра                                     | 8+ ядер                                                          |
| RAM    | 16 ГБ                                      | 32 ГБ (если PostgreSQL и Node на одном узле с другими сервисами) |
| Диск   | SSD 100 ГБ                                 | SSD/NVMe 250+ ГБ (ОС, БД, логи, временные файлы загрузок)        |
| ОС     | Linux (дистрибутив LTS) или Windows Server | По политике Заказчика                                            |


**ПО на узле приложения:**

- Node.js (LTS-версия, согласуемая с поставкой).
- PostgreSQL (актуальная стабильная ветка, поддерживаемая Заказчиком).

---

## 2. Языковая модель (LLM)

Отдельный узел **или** тот же сервер при наличии ресурсов.

### Требования к узлу LLM (GPU обязателен)


| Ресурс          | Ориентир                                                                                                                                                    |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| GPU             | NVIDIA с **не менее 16 ГБ** видеопамяти (базовый ориентир v1); **24 ГБ и выше** — если планируются более тяжёлые модели или высокая параллельность запросов |
| RAM (системная) | +16–32 ГБ к требованиям узла (зависит от ПО-обвязки LLM)                                                                                                    |
| Диск            | +50–100 ГБ под образ модели и кэш                                                                                                                           |


Ориентир по отклику при генерации JSON-правил парсинга на GPU: **секунды–десятки секунд** (зависит от модели и нагрузки).

---

## 3. Сеть

- Доступ пользователей к веб-интерфейсу из корпоративной сети (или VPN).
- Связность между сервером приложения и хостом LLM: **внутренняя сеть**, низкая задержка (лучше тот же ЦОД/VLAN).
- **Исходящий интернет** для работы системы **не обязателен** после установки моделей и обновлений по политике Заказчика.

---

## 4. Резервное копирование и отказоустойчивость

- Регулярный бэкап базы PostgreSQL — по регламенту Заказчика (снимки ВМ, `pg_dump`, и т.д.).
- Отказоустойчивость кластером PostgreSQL и несколькими узлами приложения в **v1 не входит**, unless отдельно согласовано.

---

## 5. Итоговая таблица «на быструю оценку закупки»


| Роль                                            | CPU     | RAM   | Диск       | GPU             |
| ----------------------------------------------- | ------- | ----- | ---------- | --------------- |
| Приложение + БД                                 | 8 ядер  | 32 ГБ | SSD 250 ГБ | не обязательно  |
| LLM (рекомендуется отдельно или 2-й GPU-сервер) | 4+ ядер | 32 ГБ | SSD 100 ГБ | **16+ ГБ VRAM** |


*При объединении всего на одной физической машине суммировать RAM и диск, учесть совместимость драйверов GPU с ОС.*
