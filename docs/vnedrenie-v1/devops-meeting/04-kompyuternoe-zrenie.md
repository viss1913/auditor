# Компьютерное зрение (Vision LLM) — для DevOps и ИБ

**Модуль:** `server/document_scan_llm.js`  
**Сценарий в системе:** `document_scan`  
**Режим:** on-prem, та же GPU-VM что и текстовая LLM (Ollama)

---

## 1. Зачем это нужно

Помимо **текстовой** модели (чат Martin, JSON-правила парсинга) система использует **vision-модель** — мультимодальную LLM, которая **«смотрит» на изображение/PDF** и возвращает структурированный JSON.

Это не отдельный OCR-движок (Tesseract и т.п.), а **LLM с поддержкой картинок** через OpenAI-compatible API (`/chat/completions` + `image_url` / `file`).

### Где применяется

| Сценарий | Что делает vision |
|----------|-------------------|
| **Сканы без текстового слоя** | PDF/фото договоров, актов, УПД, протоколов → таблица с колонками и полями |
| **Фото документов** | JPG/PNG → распознавание и извлечение реквизитов |
| **Брокерские PDF (ATON и др.)** | Если pdfjs не вытащил заголовки таблицы — vision определяет структуру раздела 1.2 |
| **Запрос пользователя в чате** | «Сделай таблицу: Контрагент, Дата, Сумма» → vision ищет значения по всему документу |

Если PDF **читается обычным парсером** (есть текстовый слой, ≥8 строк) — vision **не вызывается**, работает детерминированный парсинг.

## 2. Архитектура (поток данных)

```text
Пользователь загружает PDF/фото
        ↓
Node.js проверяет: есть текстовый слой?
        ↓ нет (скан)                    ↓ да
document_scan_llm.js              pdfjs / regex / сценарии
        ↓
HTTP POST → Ollama :11434/v1/chat/completions
        ↓  (модель qwen2-vl, llava, …)
JSON: headers, rows, fields, full_text
        ↓
Таблица в UI + snapshot в PostgreSQL
```

**Важно для ИБ:** на GPU уходит **фрагмент или страница документа** (base64 в теле запроса). Полный файл остаётся на VM приложения. Данные **не уходят в публичный интернет** — только на внутренний Ollama.

## 3. Модели (on-prem, Ollama)

| Ollama tag | VRAM (ориентир) | Когда выбирать |
|------------|-----------------|----------------|
| **`qwen2-vl:7b`** | ~8 ГБ | **Рекомендуем v1** — баланс качество/железо |
| `llava:13b` | ~10–12 ГБ | Альтернатива, если qwen2-vl недоступна |
| `qwen2-vl:72b` / облачные 32B | 24+ ГБ | Только при мощном GPU и требовании к качеству OCR |

**Текстовая и vision-модель** ставятся **на одну GPU-VM**. При **16 ГБ VRAM** Ollama обычно держит одну модель в памяти — text и vision **переключаются по запросу** (небольшая задержка при первом вызове). При **24 ГБ VRAM** — комфортнее параллельная работа.

Offline-установка обеих моделей: см. [03-ollama-offline.md](./03-ollama-offline.md).

## 4. Переменные окружения

```env
# Включить модуль (0 = только текстовый парсинг, без сканов)
DOCUMENT_SCAN_ENABLED=1

# Vision-модель (имя как в `ollama list`)
VISION_MODEL=qwen2-vl:7b

# URL Ollama (может совпадать с текстовой LLM)
VISION_LLM_BASE_URL=http://llm-gpu.corp.local:11434/v1

# Таймаут одного vision-запроса (сканы тяжелее текста)
VISION_TIMEOUT_MS=120000

# Порог: PDF с меньшим числом текстовых строк считается «сканом»
DOCUMENT_SCAN_MIN_LINES=8

# Брокерские PDF: vision для заголовков таблицы (1 = вкл.)
BROKER_PDF_VISION_ALWAYS=1
```

Для **локального Ollama API-ключ не нужен** (в отличие от OpenRouter/Gemini).

Шаблон prod: [deploy/.env.production.example](../../../deploy/.env.production.example).

## 5. Требования к GPU (vision отдельно от текста)

| Конфигурация | Text | Vision | Итого VRAM |
|--------------|------|--------|------------|
| Минимум v1 | qwen2.5:7b (~5 ГБ) | qwen2-vl:7b (~8 ГБ) | **16 ГБ** (по очереди) |
| Комфорт | qwen2.5:14b (~9 ГБ) | qwen2-vl:7b (~8 ГБ) | **24 ГБ** |
| Только текст (сканы выкл.) | qwen2.5:7b | — | 16 ГБ достаточно |

**На встрече с DevOps:** если заказчик загружает **много сканов PDF** — закладывать **24 ГБ VRAM** или согласовать `DOCUMENT_SCAN_ENABLED=0` на первом этапе (без vision).

## 6. Smoke test vision на стенде

```bash
# 1. Модель на GPU-VM
ollama list | grep -E 'qwen2-vl|llava'

# 2. Прямой вызов Ollama (упрощённо)
curl http://llm-gpu:11434/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"qwen2-vl:7b","messages":[{"role":"user","content":"ping"}]}'

# 3. Через приложение: загрузить PDF-скан без текстового слоя в Martin
#    → сценарий document_scan → таблица в UI
```
