# План работ по внедрению «ИИ-аудитор» v1

Документ согласуется с Заказчиком и может быть приложением к договору или внутренним регламентом проекта.

**Целевой срок внедрения v1:** **2 (два) календарных месяца** с даты выполнения условий начала работ (аванс, доступы, стенд и т.д. — как в договоре).

---

## Этап 0. Подготовка (до старта разработки/внедрения)


| Действие                                                                               | Ответственный          | Результат                 |
| -------------------------------------------------------------------------------------- | ---------------------- | ------------------------- |
| Согласование перечня форматов файлов v1 (УК Excel, отчёт брокера, PDF ДЕПО)            | Исполнитель / Заказчик | Зафиксированный список    |
| Передача обезличенных образцов или доступ к тестовым выгрузкам (при необходимости NDA) | Заказчик               | Набор файлов для проверки |
| Утверждение чек-листа приёмки                                                          | Обе стороны            | Подписанное приложение    |


---

## Этап 1. Инфраструктура Заказчика


| Действие                                                        | Результат                         |
| --------------------------------------------------------------- | --------------------------------- |
| Выделение сервера/ВМ под приложение и PostgreSQL                | Стенд готов к установке ПО        |
| При необходимости — выделение узла/GPU под языковую модель      | LLM-сервис может быть установлен  |
| Настройка сети, DNS, сертификатов (HTTPS) по политике Заказчика | Пользователи получают URL доступа |
| Согласование политики резервного копирования БД                 | Регламент бэкапов                 |


---

## Этап 2. Установка программного комплекса


| Действие                                                       | Результат                        |
| -------------------------------------------------------------- | -------------------------------- |
| Установка PostgreSQL, создание БД, применение миграций         | Рабочая схема данных             |
| Развёртывание серверной части (Node.js) как сервиса            | API доступен во внутренней сети  |
| Сборка и публикация веб-клиента                                | Интерфейс открывается в браузере |
| Настройка переменных окружения (БД, URL локальной LLM, прочее) | Конфигурация «прод»              |


---

## Этап 3. Локальная языковая модель


| Действие                                                      | Результат                                 |
| ------------------------------------------------------------- | ----------------------------------------- |
| Установка выбранного ПО (например Ollama или аналог) и модели | Сервис LLM отвечает по API                |
| Проверка генерации JSON-правил на реальных превью Excel       | Стабильный формат ответа для парсера      |
| Настройка производительности GPU и модели                     | Приемлемое время ответа для пользователей |


---

## Этап 4. Интеграционное тестирование


| Действие                                                                  | Результат                             |
| ------------------------------------------------------------------------- | ------------------------------------- |
| Загрузка данных по всем трём источникам                                   | Данные в БД, корректный просмотр в UI |
| Сценарий с нестандартным УК: запрос пользователя + ИИ-правило + загрузка  | Сделки УК распознаны по правилу       |
| Запуск аудита, проверка фильтров и выгрузки CSV                           | Результат совпадает с чек-листом      |
| Проверка отсутствия ухода данных во внешние API (по требованию Заказчика) | Подтверждение локального контура      |


---

## Этап 5. Приёмка


| Действие                                          | Результат                            |
| ------------------------------------------------- | ------------------------------------ |
| Совместный проход по чек-листу                    | Протокол / отметки                   |
| Передача инструкций администратору и пользователю | Комплект документов                  |
| Подписание акта сдачи-приёмки                     | Основание для окончательного платежа |


---

## Этап 6. Обучение


| Действие                             | Результат                                      |
| ------------------------------------ | ---------------------------------------------- |
| Занятия с ключевыми пользователями   | Навыки загрузки, ИИ-парсинга, аудита, экспорта |
| При необходимости — запись/шпаргалка | Материалы для внутреннего использования        |


---

## Зависимости и риски

- Сроки этапов 2–4 зависят от своевременной готовности железа и доступа администраторов Заказчика.
- Задержка поставки GPU откладывает ввод блока ИИ в эксплуатацию (сроки этапа с LLM привязать к готовности железа).
