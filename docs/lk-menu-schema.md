```
┌─────────────────────────┐
```

## Разделы и подразделы

| Раздел | Подразделы | Содержимое |
|--------|------------|------------|
| **AI Martin** | — | **Первый экран при входе.** Сообщение от Мартина (приветствие), поле ввода текста для диалога. Потом: файл + запрос → правило → парсинг. |
| **Аудит ОПИФ** | Исходные файлы | Заглушка (потом: список загруженных файлов, хранилище) |
| | УК | Загрузка, парсинг, таблица сделок УК |
| | Брокер | Загрузка, парсинг, таблица сделок брокера |
| | ДЕПО | Загрузка, парсинг, таблица сделок ДЕПО |
| **Аудит договоров** | — | Заглушка: раздел под будущий аудит договоров. |
| **Аудит сделок** | — | Заглушка (потом: сверка УК↔Брокер↔ДЕПО — подготовка, запуск, фильтры, таблица результатов, экспорт). |
| **Выход** | — | Внизу сайдбара |

## Виды аудита на рынке (справочно)

Краткий справочник — какие виды аудита бывают на рынке (для лендинга или заглушек):

| Вид | Описание |
|-----|----------|
| **Внешний аудит** | Независимая проверка отчётности (аудиторская фирма). Для акционеров, регуляторов. |
| **Внутренний аудит** | Проверка процессов и рисков внутри компании. |
| **Финансовый аудит** | Проверка финансовой отчётности на соответствие МСФО/РСБУ. |
| **Налоговый аудит** | Проверка правильности расчёта и уплаты налогов (в т.ч. по инициативе ФНС). |
| **Операционный аудит** | Оценка эффективности процессов, издержек, рисков. |
| **Аудит соответствия (compliance)** | Соблюдение законов, регламентов, внутренних политик. |
| **IT-аудит** | Проверка ИТ-инфраструктуры, безопасности, доступов. |
| **Аудит договоров** | Проверка договоров на соответствие регламентам, выявление рисков, полнота условий. |
| **Аудит сделок** | Проверка и сверка сделок (в т.ч. УК↔Брокер↔ДЕПО — наш продукт). |

В интерфейсе — один экран-заглушка с этим списком и короткими описаниями.

## Навигация по приложению

```
Лендинг (главная)
    │
    ├── [Личный кабинет] ──► Вход (email + пароль, заглушки)
    │                            │
    │                            └──► Личный кабинет
    │                                 Первый экран = AI Martin (Мартин пишет + поле ввода)
    │
    └── Остаёмся на лендинге
```

## Область контента (справа от sidebar)

- **AI Martin** — при входе в ЛК открывается этот раздел. Сверху: сообщение от Мартина (приветствие). Ниже: поле ввода текста. Дальше по сценарию: загрузка файла, запрос правил, генерация, парсинг.
- **Аудит ОПИФ** → Исходные файлы (заглушка) / УК / Брокер / ДЕПО / Аудит — УК, Брокер, ДЕПО: как сейчас (зона загрузки + таблица + поиск, пагинация, CSV). Аудит — запуск сверки, результаты.
- **Аудит договоров** — заглушка (экран «В разработке» или описание продукта).
- **Аудит сделок** — заглушка (потом: «Проверить ключи», «Запустить аудит», фильтры, таблица результатов, экспорт).
