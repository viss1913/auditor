# Аудит Асоль — Документация

Система сверки сделок по трём источникам: **УК**, **Брокер**, **ДЕПО**.

---

## Запуск системы

### 1. Сервер (запускать первым)
```powershell
cd c:\Users\User\Desktop\auditor_3\server
node index.js
```
Сервер поднимается на `http://localhost:3001`. Успешный старт:
```
Сервер Асоль на порту 3001
✅ База данных готова (таблицы проверены)
```

### 2. Фронтенд
В отдельном терминале (из корня проекта):
```powershell
npm run dev
```
Открыть в браузере: `http://localhost:5173`

> **Важно:** сервер всегда запускать из папки `server/`, иначе не найдёт зависимости.

## Загрузка данных

В интерфейсе три вкладки: **УК**, **Брокер**, **ДЕПО**.

### УК (Карточка счёта)
- Формат: Excel (.xls, .xlsx, .xlsm)
- Кнопка "Выбрать файлы вручную"
- Ищем проводки: Дт **58.01** / Кт **76**
- Парсер читает дату, название бумаги, рег. номер, кол-во, сумму

### Брокер (Отчёт брокера)
- Формат: Excel, файлы с именем `1F018_*`
- Кнопка **📁 Выбрать папку** — автоматически фильтрует только `1F018_*` файлы из всех подпапок
- Парсер ищет данные в **Разделе 1.2** ("сделки, которые не исполнены")
- Извлекает: дату, тип операции, бумагу, рег. номер, ISIN, кол-во, сумму, валюту, дату регистрации, комиссии

### ДЕПО (Выписка о движении ЦБ)
- Формат: PDF
- Выбрать все PDF-файлы выписок
- Парсер читает: название бумаги, рег. номер / ISIN, дату операции, тип (Зачисление/Списание ЦБ), количество

### Режимы загрузки
- **Заменить всё** — удаляет старые данные источника и загружает новые
- **Добавить к текущим** — дополняет существующие данные

## Поиск и экспорт

На каждой вкладке:
- 🔍 **Строка поиска** — фильтр по названию, ISIN, рег. №, дате, виду сделки
- ⬇ **CSV** — выгрузить текущую выборку в Excel-совместимый CSV (с BOM)
- Данные сортируются по дате по возрастанию автоматически

## Аудит

Вкладка **🔍 Аудит** → кнопка **▶ Запустить аудит**.

### Логика проверки

Берём каждую строку из **УК** и проверяем два условия:

#### УК ↔ Брокер
Ищем строку у брокера где совпадают:
| Поле | УК | Брокер |
|------|-----|--------|
| Дата регистрации | `Дата рег.` | `Дата рег.` |
| Бумага | `Рег. №` или `ISIN` | `Рег. №` или `ISIN` |
| Количество | `Кол-во` | `Кол-во` |
| Сумма | `Сумма` | `Сумма` (±1 руб.) |

> Дата сделки у брокера может быть на 1–2 дня раньше (T+1/T+2), сравниваем именно по **дате регистрации**.

#### УК ↔ ДЕПО
ДЕПО хранит суммарное количество бумаг за день. Поэтому:
1. Суммируем `Кол-во` по всем строкам УК с одной бумагой на одну дату регистрации
2. Ищем в ДЕПО запись где совпадают: дата + бумага + **суммарное** кол-во

### Результат

Таблица показывает по каждой строке УК:

| Колонка | ✅ Найдено | ❌ Не найдено |
|---------|-----------|--------------|
| УК↔Брокер | найдена сделка + дата из отчёта брокера | нет совпадения |
| УК↔ДЕПО | найдена регистрация + дата | нет совпадения |

Строки **подсвечиваются зелёным** если оба источника найдены, **красноватым** — если хоть один не нашёлся.

### Фильтры аудита
- **Все** — все строки
- **❌ Проблемы** — только строки где что-то не нашлось
- **✅ Совпало** — только полностью сошедшиеся строки

Результат аудита тоже выгружается в **CSV**.

## Карта файлов: кто за что отвечает

### Парсинг (стандартный)

Основная логика парсинга живёт в `server/index.js` — три функции:

| Функция | Что парсит | Формат | Ключевая логика |
|---------|-----------|--------|-----------------|
| `parseUK()` | Карточку счёта УК | Excel (.xls/.xlsx/.xlsm) | Ищет проводки Дт 58.01 / Кт 76. Строка с «Кол.» дополняет предыдущую запись количеством и суммой |
| `parseBroker()` | Отчёт брокера | Excel (файлы `1F018_*`) | Ищет Раздел 1.2, извлекает дату, тип операции, бумагу, ISIN, рег.№, кол-во, сумму, валюту, дату регистрации, комиссии |
| `parseDepo()` | Выписку ДЕПО | PDF | Через pdf-parse извлекает текст, ищет название бумаги, рег.№/ISIN, дату, тип (Зачисление/Списание ЦБ), количество |

### Парсинг (умный / ИИ)

| Файл | Что делает |
|------|-----------|
| `server/smart_parse_uk.js` | Функция `smartParseUK()` — парсит Excel УК по JSON-правилам (debit_account, credit_account, date_start, date_end). Используется когда при загрузке передаётся `ruleJson` |
| `server/ai_parser_api.js` | API для работы с Qwen (OpenRouter). `getExcelPreview()` берёт первые 20 строк Excel, отправляет в нейросеть, получает JSON-правило для `smartParseUK`. Эндпоинт: `POST /api/ai/generate-rule-from-file` |

### Аудит

Вся логика аудита тоже в `server/index.js` — два эндпоинта:

| Эндпоинт | Что делает |
|----------|-----------|
| `GET /audit/preview` | Подготовка к аудиту: считает записи УК/Брокер/ДЕПО, строит ключи (дата\|бумага\|buy/sell), показывает пересечения и расхождения между источниками |
| `GET /audit` | Сам аудит: для каждой сделки УК ищет совпадение у Брокера (по дате, бумаге, кол-ву, сумме ±1 руб.) и у ДЕПО (по группе дата + бумага + тип, сравнивает суммарное кол-во) |

### Отладочные / тестовые скрипты

| Файл | Категория | Описание |
|------|-----------|----------|
| `server/debug_matching.js` | Аудит | Отладка матчинга: выбирает сделки по reg_number/security_name, выводит UK/Broker/DEPO, ключи и совпадения |
| `server/debug_audit.js` | Аудит | Отладка аудита: проверяет формат дат и кол-ва для конкретной бумаги в УК и ДЕПО |
| `server/test_depo_script.js` | Парсинг | Копия `parseDepo()` для тестов парсинга PDF |
| `server/test_fix.js` | Парсинг | Тест парсинга УК с альтернативным маппингом колонок |
| `server/test_pdf.js` | Парсинг | Тест извлечения текста из PDF через pdf-parse |
| `server/dump_uk_structure.js` | Парсинг | Дампит первые 50 строк Excel УК для анализа структуры |

### Утилиты и БД

| Файл | Описание |
|------|----------|
| `server/migrate_db.js` | Миграция БД: создаёт таблицы `projects`, `parsing_rules`, добавляет `project_id` в `trades` |
| `server/check_uk_data.js` | Проверка данных УК: ищет нулевые суммы/кол-во, дубликаты |
| `server/check_db.js` | Проверка подключения к PostgreSQL |
| `server/sum_uk.js` | Считает количество и сумму сделок УК |
| `server/discover_api.js` | Проверяет экспорты библиотеки pdf-parse |

### Фронтенд

| Файл | Описание |
|------|----------|
| `src/App.jsx` | Главный React-компонент: вкладки УК/Брокер/ДЕПО/ИИ/Аудит, загрузка файлов, запуск парсинга и аудита, экспорт CSV, поиск, пагинация |
| `src/main.jsx` | Точка входа React |
| `src/index.css` | Глобальные стили (тема, таблицы, кнопки, подсветка аудита) |

### Сводка: где искать логику

```

АУДИТ                →  server/index.js  (GET /audit, GET /audit/preview)
UI                   →  src/App.jsx
```

## Техническая информация

| Компонент | Технология |
|-----------|-----------|
| БД | PostgreSQL (таблица `trades`) |
| Сервер | Node.js + Express |
| Фронтенд | React + Vite |
| PDF парсер | pdf-parse v1 |
| Excel парсер | xlsx |

### Структура БД (таблица `trades`)
```
period            DATE        — дата сделки
operation_type    TEXT        — Покупка / Продажа / Зачисление ЦБ / ...
security_name     TEXT        — название бумаги
reg_number        TEXT        — регистрационный номер выпуска
isin              TEXT        — ISIN код
quantity          NUMERIC     — количество
amount            NUMERIC     — сумма
currency          TEXT        — валюта
registration_date TEXT        — дата регистрации (T+1/T+2)
fee               NUMERIC     — комиссии брокера
source            TEXT        — 'UK' | 'Broker' | 'DEPO'
```
